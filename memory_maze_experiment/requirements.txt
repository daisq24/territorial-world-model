# Version pinning notes (learned the hard way):
# - mujoco 3.2.x renamed MjModel struct fields (flex_interp, tex_rgb) — breaks
#   older dm_control. mujoco>=3.3 drops macOS Py3.9 wheels. 3.1.6 is the sweet
#   spot: has prebuilt wheels and the old field names dm_control expects.
# - dm_control 1.0.24+ references struct fields that don't exist in 3.1.x
#   (cam_orthographic etc.). 1.0.18 / 1.0.20 are the last safe ones for 3.1.x.
# - NumPy 2.0 changed np.array(copy=False) to hard-error, which dm_control's
#   entity.py hits on reset(). So we cap numpy at <2.0.
mujoco>=3.1.4,<3.2
dm-control>=1.0.18,<1.0.21
gym>=0.26,<1.0
numpy>=1.23,<2.0
matplotlib>=3.6,<3.10  # 3.10 requires numpy>=2.0
scipy>=1.10,<1.14  # 1.14+ requires numpy>=2.0
